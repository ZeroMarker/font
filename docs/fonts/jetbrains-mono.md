# JetBrains Mono

JetBrains Mono 是 JetBrains 推出的开源等宽字体，面向开发者的代码阅读与编写场景。它常见于 IDE、编辑器、终端和技术文档中的代码块。

## 定位

JetBrains Mono 的核心目标是提升代码可读性。它不是一般的正文等宽字体，而是针对开发者长期阅读代码的字符识别、行距节奏、符号清晰度和连字体验进行设计。

## 风格特征

- 等宽设计，适合代码对齐。
- 字符区分度较高，便于识别易混淆字符。
- 提供代码连字，能把部分操作符显示为更紧凑的组合形态。
- x-height 较高，小字号下阅读稳定。
- 西文代码体验优秀，但中文需要搭配 fallback 字体。

## 适用场景

- JetBrains IDE、VS Code、终端和代码编辑器。
- 技术博客、文档站和演示稿中的代码块。
- 需要清晰展示操作符、括号和数字的场景。
- 长时间写代码的个人开发环境。

## 不适合场景

- 中文正文或中英文长段落阅读。
- 品牌标题、海报和营销视觉。
- 数据表格之外的大面积普通 UI 文案。
- 不希望启用连字的精确字符教学场景。

## 排版建议

在编辑器中可从 13px 到 15px 测试字号，并根据系统渲染调整行高。若连字影响代码教学或符号逐字符识别，可以关闭 ligatures。中文注释较多时，应设置稳定的中文 fallback，例如更纱黑体、Noto Sans Mono CJK 或系统中文等宽字体。

技术文档中，JetBrains Mono 适合代码块和内联代码，不建议作为正文主字体。代码块背景、行高和字重需要一起测试，避免字符过密。
