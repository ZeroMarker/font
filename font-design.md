# 字体设计指南

本文用于记录字体设计的基础概念、工作流程和交付检查项，适合在设计自有字体、选择品牌字体或制定产品字体规范时参考。

## 设计目标

开始设计前先明确字体要解决的问题：

- 使用场景：品牌标题、正文阅读、移动界面、代码编辑、数据大屏、包装或印刷。
- 文字系统：中文、拉丁、西里尔、日文、韩文、符号、数字和图标。
- 气质关键词：现代、可靠、温和、技术感、手写感、传统、几何或人文。
- 阅读距离：近距离长文阅读、屏幕小字号、远距离导视或大标题展示。
- 交付范围：字重数量、是否需要斜体、是否需要可变字体、是否需要 Web 字体。

清晰的目标可以帮助后续判断笔画、字面、字重和间距是否合理。

## 基础术语

- 字面：字符在字框内实际占用的视觉面积。中文字体通常需要控制字面一致性。
- 字重：笔画粗细层级，如 Light、Regular、Medium、Bold。
- 字宽：字符横向比例，如 Condensed、Normal、Expanded。
- x-height：拉丁小写字母主体高度，影响西文小字号可读性。
- ascender / descender：拉丁字母上伸部和下伸部。
- baseline：基线，拉丁字符通常落在基线上。
- kerning：特定字符组合的间距微调。
- tracking：整段文字统一增减字距。
- hinting：针对屏幕栅格显示的提示信息，影响小字号清晰度。
- glyph：一个字形图像，可能对应一个或多个字符。

## 字体结构

### 中文

中文字体设计重点在结构统一和重心稳定：

- 横、竖、撇、捺、点、钩等基础笔画应有统一规则。
- 左右结构、上下结构、包围结构和独体字需要保持视觉重心一致。
- 常用部件应复用设计规则，但不能机械复制，需要按字形微调。
- 标点、全角符号和数字要与中文节奏匹配。
- 大字面字体适合屏幕和标题，小字面字体在长文中更稳重。

### 拉丁

拉丁字体设计重点在节奏、曲线和字间关系：

- 先设计 `n`、`o`、`H`、`O` 等关键字符建立宽度和曲线规则。
- 小写的 x-height 会明显影响可读性和字体气质。
- 圆形字符需要视觉超出基线和高度线，避免看起来偏小。
- `A V W Y T` 等字符组合通常需要 kerning。
- 数字需要根据场景选择比例数字、等宽数字、表格数字或旧式数字。

## 设计流程

1. 定义需求：明确字体用途、覆盖字符集、授权方式和交付格式。
2. 收集参考：分析相近风格字体、品牌视觉、竞品和实际使用样张。
3. 建立核心字形：中文可先做基础笔画、偏旁和高频字；拉丁可先做核心大小写与数字。
4. 制定规则：确定笔画端点、转角、曲线、重心、字宽、字面和标点风格。
5. 扩展字符集：按高频字符、产品必需字符和完整字符集逐步扩展。
6. 调整间距：先设 side bearing，再处理 kerning 和中西文混排。
7. 生成测试文件：导出 OTF、TTF、WOFF2 或可变字体文件。
8. 样张验证：在真实页面、移动端、印刷稿和代码环境中检查。
9. 修正问题：重点修正重心、灰度、糊字、缺字、标点和字符混淆。
10. 版本交付：记录版本、字符集、授权、变更日志和已知限制。

## 字重与可变字体

多字重设计要保持家族一致性：

- 先确定 Regular 与 Bold，再推导 Light、Medium、SemiBold 等中间字重。
- 细字重要避免断笔，粗字重要避免笔画粘连和字腔堵塞。
- 中文粗字重尤其要检查复杂字，如“赢”“鬱”“囊”等。
- 可变字体需要定义轴，例如 `wght`、`wdth`、`slnt`、`opsz`。
- 可变字体在 Web 中能减少文件数量，但需要测试浏览器兼容和渲染质量。

## 中西文混排

中西文混排常见问题包括字号不协调、基线不齐、标点拥挤和数字风格不统一。

建议检查：

- 中文与拉丁的视觉高度是否接近。
- 英文缩写、产品名和数字在中文段落中是否突兀。
- 半角标点和全角标点是否符合排版规范。
- 等宽代码字体与中文 fallback 混排时行高是否稳定。
- 表格中的数字是否应使用等宽数字。

## 屏幕显示

屏幕字体要优先考虑小字号清晰度：

- 低分辨率或小字号场景下，过细笔画容易发虚。
- 字腔过小会导致字符糊成一团。
- UI 字体应保留明确的字重层级，避免 Regular 与 Medium 区分不明显。
- Web 字体应使用 `font-display` 策略，降低加载失败或闪烁影响。
- 移动端需要分别检查 iOS、Android 和不同浏览器的渲染差异。

## 文件格式

| 格式 | 适用场景 | 说明 |
| --- | --- | --- |
| OTF | 设计交付、印刷、桌面安装 | 支持 OpenType 特性，常用于专业字体。 |
| TTF | 桌面安装、部分系统兼容 | 兼容性好，屏幕显示常见。 |
| WOFF | Web 字体 | 面向网页加载。 |
| WOFF2 | Web 字体 | 压缩率更高，现代浏览器优先选择。 |
| VF | 可变字体 | 一个文件承载多个轴或字重变化。 |

## 质量检查

交付前建议至少检查以下内容：

- 字符覆盖：是否包含产品所需字符、标点、数字、货币符号和品牌词。
- 字形一致性：同类笔画、端点、转角、曲线和部件是否统一。
- 灰度均匀：段落整体黑白分布是否平衡，是否有明显过黑或过亮字符。
- 间距稳定：常见词组、数字、英文缩写和标点组合是否拥挤或松散。
- 小字号可读性：12px、14px、16px 等常见屏幕字号是否清楚。
- 大字号表现：标题和海报尺寸下曲线、端点和细节是否经得起放大。
- 跨平台渲染：macOS、Windows、iOS、Android、Chrome、Safari 和 Firefox 显示是否可接受。
- 文件信息：字体名称、版本号、版权信息、许可证和 family naming 是否正确。

## 授权与合规

字体授权是设计交付的一部分，不应在项目后期才处理。

- 确认是否允许商用。
- 确认是否允许嵌入 App、网页、PDF、电子书或硬件设备。
- 确认是否允许修改、二次分发或作为品牌字体发布。
- 记录字体来源、版本和许可证链接。
- 品牌项目应保留授权文件、购买记录或开源许可证副本。

## 常用工具

- Glyphs：适合字体绘制、OpenType 特性和字体家族管理。
- FontLab：专业字体设计和工程工具。
- RoboFont：适合脚本化和定制字体工作流。
- FontForge：开源字体编辑工具。
- fonttools：字体检查、转换和自动化处理工具。
- ttx：将字体二进制与 XML 表示互转，便于检查字体表。

## 样张建议

测试字体时应准备覆盖真实场景的样张：

- 中文短标题、长标题和正文段落。
- 英文大小写、数字、货币、日期和技术缩写。
- 中英混排段落。
- 表格、按钮、标签、导航、输入框和错误提示。
- 代码片段、终端输出和日志。
- 极端字符，如复杂汉字、窄字母、宽字母和易混淆字符。

## 交付清单

- 字体文件：OTF、TTF、WOFF2 或可变字体。
- 字符集说明：覆盖范围、缺字列表和计划扩展范围。
- 样张文件：屏幕、印刷、标题、正文和混排样张。
- 使用规范：推荐字号、字重、行高、字距和 fallback 字体链。
- 授权文件：许可证、版权声明和使用限制。
- 版本记录：版本号、发布日期、变更内容和已知问题。
